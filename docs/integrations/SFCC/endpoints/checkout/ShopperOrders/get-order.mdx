# Get Order

## Purpose

This endpoint is used to fetch a customer order by order number.

## Example

<Tabs>
  <TabItem value="commerce-sdk" label="Commerce SDK">
    ```js
    const parameters = {
      locale: "en_CA",
      orderNo: "ABC10001", // required
    }
    const orders = await shopperOrdersClient.getOrder({ parameters });
    ```
  </TabItem>
  <TabItem value="scapi" label="SCAPI">
    ```http
    GET /checkout/shopper-orders/v1/organizations/{organizationId}/orders/{orderNo}?siteId=RefArch HTTP/1.1
    Host: {shortcode}.api.commercecloud.salesforce.com:443
    ```
  </TabItem>
</Tabs>

## Request

### URL

`GET https://{shortCode}.api.commercecloud.salesforce.com/checkout/shopper-orders/{version}/organizations/{organizationId}/orders/{orderNo}?siteId=RefArch`

### Headers

- `Authorization: Bearer {slas_token}`

### URL Parameters

- `shortCode` _string_: A region-specific merchant identifier. **(required)**
  - Retrieved from Business Manager > Administration > Salesforce Commerce API Settings > Shortcode
  - Example: `0dnz6oep`
- `version` _string_: API version identifier. **(required)**
  - Example: `v1`
- `organizationId` _string_: The identifier for the organization. **(required)**
  - Retrieved from Business Manager > Administration > Salesforce Commerce API Settings > organization ID
- `orderNo` _string_: The ID of the requested order **(required)**
  - Example: `ABC10001`

### Query Parameters

- `siteId` _string_: The site context of the request. Used to handle site-specific category attribute values. **(required)**
  - Example: `RefArch`
- `locale` _string_: The locale of the current request (used for category localization)
  - Example: `fr_CA`

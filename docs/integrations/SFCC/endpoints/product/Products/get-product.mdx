# Get Product

## Purpose

This endpoint is used to fetch a single product by ID using the Admin API.

## Example

<Tabs>
  <TabItem value="commerce-sdk" label="Commerce SDK">
    ```js
    const parameters = {
      productId: "25328702M"
    }
    const products = await productsClient.getProduct({ parameters });
    ```
  </TabItem>
  <TabItem value="scapi" label="SCAPI">
    ```http
    GET /product/products/v1/organizations/{organizationId}/products/{productId} HTTP/1.1
    Host: {shortcode}.api.commercecloud.salesforce.com:443
    ```
  </TabItem>
</Tabs>

## Request

### URL

`GET https://{shortCode}.api.commercecloud.salesforce.com/product/products/{version}/organizations/{organizationId}/products/{productId}?siteId=RefArch`

### Headers

- `Authorization: Bearer {access_token}`

### URL Parameters

- `shortCode` _string_: A region-specific merchant identifier. **(required)**
  - Retrieved from Business Manager > Administration > Salesforce Commerce API Settings > Shortcode
  - Example: `0dnz6oep`
- `version` _string_: API version identifier. **(required)**
  - Example: `v1`
- `organizationId` _string_: The identifier for the organization. **(required)**
  - Retrieved from Business Manager > Administration > Salesforce Commerce API Settings > organization ID
- `productId` _string_: The ID of the requested product **(required)**
  - Example: 25328702M

### Query Parameters

- `siteId` _string_: The site context of the request. Used to handle site-specific product assignments and attribute values. **(required)**
  - Example: `RefArch`
